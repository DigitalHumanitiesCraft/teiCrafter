# Milestone evaluation: M2.5 annotation visibility layer and M2.6 inline cell action chooser

Date: 2026-06-10. Milestone: the editor UI restructuring decided 2026-06-09
(Editopia plan AP-A and AP-B): annotations become visible in the reading text,
and every cell operation becomes reachable at the cell without pre-toggled
toolbar modes. This report is the repo-side record alongside the journal
entry of the same date.

## 1. What the milestone promised

From the implementation plan (AP-A, AP-B): linked mentions render in their
entity-type colour family with a tooltip naming the entity; notes get a
triple-coded marker (colour, icon, position); a legend strip names the visual
codes; the engine model exposes a read-only `cell.mention` projection.
Clicking a cell opens an in-place action chooser (edit, note, mark, link via
entity picker, cancel) following the proven `.ed-crit-pick` pattern; the
toolbar toggles remain as shortcuts; the index-initiated link flow is
preserved. Everything token-only, label-consistent, and lossless.

## 2. How it was produced

The engine change came first: `mentionRef(node)` in `edition.js` walks the
parent chain to the nearest `<name ref>` (through critical wrappers, stopping
at structural elements), giving every text cell a `mention` field; gap cells
project `null`. The renderer (`editor-app.js`) builds one entity-meta map per
render and assigns `mention-<kind>` classes; the kind comes from the entity
TYPE that `readEntities` reports, not the id prefix, so hand-authored ids
colour correctly. The chooser (`beginActions`) replaced the bare edit-input
default; all inline mutations were unified into a shared `applyDocFn` helper
honouring the engine's SAME-doc no-op contract. Two token pairs
(`--color-workName`, `--color-eventName`) were added in `style.css`; the
editor stylesheet stays raw-hex-free.

## 4. Deviations and judgement calls

1. Mention kind from entity type instead of the planned id prefix (superset
   of the plan; prefix-less ids still colour).
2. The temporary index-selection highlight moved to its own class
   (`mention-hit`) so clearing it cannot strip the permanent visibility
   classes; a necessary consequential change not in the plan text.
3. Mentions of AI-proposed, unconfirmed entities render in the violet AI
   family with a tooltip note; an addition mandated by the design system's
   separability rule rather than by AP-A.
4. Double-click as quick edit works through the chooser (`e.detail` guard on
   non-edit buttons, box `dblclick`, edit focused for plain Enter).
5. One cosmetic defect found and fixed during browser verification: link
   status messages now trim the cell text.

## 5. Open after this milestone

A six-lens adversarial review of the diff completed its finder phase (the
verification phase hit a session limit; findings were triaged manually
afterwards). Confirmed findings are handled as a separate follow-up package
on top of this milestone, notably: legend chips render colourless (every
visual-code CSS rule is scoped to `.ed-w`), relinking a critically-wrapped
mention nests `<name>` inside `<name>`, an Escape-listener leak when a
chooser is displaced by an external render, missing `e.detail` guards in the
entity picker, and cancel triggering a full re-render that resets scroll and
zoom. See the follow-up commit and journal entry.

Operator gates: sight-off of both UIs, push (branch is ahead of origin),
and the M2.5/M2.6 entries in goals.md stay closed only if the operator
accepts the as-built deviations recorded in design.md.

## 6. Verdict

Milestone met as scoped: both workstreams are built, token-only and
label-consistent, the engine projection is covered by a dedicated offline
check, the full regression is green, and both real objects round-trip
byte-identically in the browser. The adversarial review found real polish
and robustness defects (none breaking losslessness), routed into a follow-up
package rather than reopening the milestone.
